Unicode and UTF-8

0.0.1 ASCII码的诞生

美国人最先发明了电脑，电脑是由二进制存储的，故美国人至少要表示a-Z，A-Z，0-9，!@#$%^&*()等等字符，最终凑够了128种符号

定义1字节为8位二进制数，可以表示2^8=256个字符；

故ASCII码就诞生了，只占用1字节，且第一位为1，后七位分别对应a-z，A-Z，0-9，!@#$%^&*()等等字符

0.1.1 中国的诞生和GB2312字符集

计算机传入中国，故显示汉字成为了一个新的问题，由于汉字有几千个，故用1字节表示不了，所以我们改用了2字节，这样，我们可以表示2^16=65536个字符，经多方搜查，最终确定了汉字6763个和非汉字图形字符682个，共计7445个字符，于是便有许多空出来的位置，那么专家是怎么做的呢？

Q1. 计算机怎么确认是ASCII码还是GB2312?
A1. 我们发现ASCII码总是0xxx xxxx的形式，所以GB2312的一定是1xxx xxxx 1xxx xxxx的形式，这样算来GB2312就只能存2^14=16384个字符，不过这也绰绰有余了。

0.1.2 中国的发展和GBK字符集

我们发现在GB2312中1xxx xxxx 0xxx xxxx没有使用，即有2^14=16384个字符没有使用，于是加以扩充，形成了GBK字符集

1.0.1 国际大一统和Unicode

不仅仅中国有中国的字符集，世界上各个国家都有自己的字符集，于是Unicode站出来了，并给出了如下的方案

方案1：我们使用4字节存储一个字符，这样就可以表示2^32=42 9496 7296个字符啦，一个一个编码就可以啦
比如“严”字的UNICODE编码为00004E25,00000000 00000000 01001110 00100101

那么这样会有两个问题

Q1.纯英文文件大了4倍

Q2.与ASCII不兼容，即看到00004E25是翻译成“严”还是“\0\0N%”

为了解决这个问题，我们决定对UNICODE加以改造，这就是UTF-8

方案2：（UTF-8字符集）

UNICODE：0000 0000-FFFF FFFF（十六进制）4字节

UTF-8: 1字节-6字节

git的使用方法

工作区 working repository
暂存区 stage (Index)
仓库 repository (Head)
具体讲解可参看B站小甲鱼git系列

Move hello.cpp to local repo

Step 1

open git bash in the diresctory

Step 2

git init
to create a hidden folder named .git

Step 3

git add hello.cpp
to move hello.cpp to Stage
git add .
move all files to Stage

Step 3.1

git status
you can use this to check the status of the hello.cpp

Step 4

git commit -m 'this is a hello cpp'
to move hello.cpp to local repo in git

Step 4.1

git status
you can use this to check the status of the hello.cpp

Delete hello.cpp in git

Step 1

rm -rf hello.cpp
to delete hello.cpp in working space

Step 2

git rm hello.cpp
to delete hello.cpp in Stage and working space

Step 3

git commit -m 'delete hello.cpp'
to tell others why you commit your work

Move hello.cpp from local repo to repo in github

Step1

git push

Check the files in repo

use git log to check the files

use git reflog to check all historical files

Reset命令

git reset -- mixed HEAD~ 等效于
git reset -- HEAD~
相当于撤销add+commit

git reset --soft HEAD~
相当于撤销commit

git reset --hard HEAD~

相当于撤销add+commit，还删除了工作区文件

番外：Reset命令原理

git reset -- mixed HEAD~移动HEAD指针，还将之前repo中的文件覆盖现在的stage

git reset -- soft HEAD~只移动HEAD指针

git reset -- hard HEAD~移动HEAD指针，还将之前repo中的文件覆盖现在的stage和working repo

比较

比较工作区和暂存区

git diff

比较暂存区和repository

git diff --staged

比较工作区和repository

git diff HEAD

比较工作区和history repository

git diff 2d4efb（此为ID）

比较history repository1和history repository2

git diff 2d4efb（此为ID1） 2d498b（此为ID2）

分支

创建分支feature

git branch feature

切换分支

git checkout feature

查看当前分支

git log --decorate

合并分支

git merge feature

删除分支

git branch -d feature

light_physics

双缝干涉实验

A:对于光的相干条件，频率相等和相位差恒定不是一个意思吗？

Q:对于光波来说，是不一样的。因为光是由于原子的跃迁产生的，而且发出光的时间很短，即发出的是一个个有限波长的光波片段。即使两波源频率相等，且在某一点相遇的两列波片段1和2能相互加强(相位差恒定），但不能保证下一时刻的两列波在该点也是加强，因为即使两相干波中也有加强点和减弱点，即可能下一时刻该点可能是减弱点或是都不是，也就是说在该点的振动不是始终加强，即不能发生相干。注意，书上所说的波源的相位差要恒定，指的是在任意时刻发出的两列光相位差要相等，也是指在空间某一点任意某一时刻相位差恒定。而光发出的波列是不满足这一要求的。

python不同版本的兼容问题（终极版）

让python2.7，python3.6，python3.8其乐融融

第一步

安装python2.7，python3.6，python3.8,安装时已经有pip,并且默认添加到环境变量PATH

第二步

将所有需要python的命令分别改成py -3.8或py -3.6或py -2.7

例如：
改前的cmd命令：python pip install ***
改后的cmd命令：py -3.8 pip install ***

PyUserInput的安装和使用

第一章

PyUserInput的安装

简介：

PyUserInput为github上的一个开源项目，网址为https://github.com/PyUserInput/PyUserInput,可以在开始一切之前先阅读此网页,有一个大概的了解,然后本文将对网页中的内容加以补充.

第一步：

下载python2.7版本，配置好环境变量,确保pip已经设置好

第二步：

在cmd中运行pip install wheel

第三步:

在cmd中运行pip install pywin32

第四步:

在https://www.lfd.uci.edu/~gohlke/pythonlibs/中找到pyHook‑1.5.1‑cp27‑cp27m‑win_amd64.whl并下载到某路径(以E:\pyhook为例)

第五步:

在cmd中运行pip install E:\pyhook\pyHook‑1.5.1‑cp27‑cp27m‑win_amd64.whl

第六步:

在cmd中运行pip install PyUserInput

大功告成！

注意：

1.本文只适用于python2.7版本

2.按照与本文类似的方法,也可以适用于python3.4-3.7版本

3.如果电脑中既有python2又有python3会比较麻烦,需要更多的操作,本文不在此赘述,在另一篇文章中会讲解

4.本文所演示的为windows64位系统以及64位python2.7
